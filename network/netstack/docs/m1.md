# M1：UDP echo + channel link

本文档是 C++ 重写 **M1** 阶段的实施指南，对应 [`plan.md`](../plan.md) / [`todo.md`](../todo.md) 中「UDP echo」里程碑。  
M0 地基见 [`docs/m0.md`](m0.md)；架构背景见 [`docs/refer-arch.md`](refer-arch.md)。

**总原则**：接口形状跟 Netstack（`references/`），实现深度按 smoltcp 裁剪；**测试驱动 channel link**，不依赖 TUN、不依赖完整 Linux socket 语义。

---

## 定位

M1 在 M0 之上接通 **传输层 demux + UDP 最小 endpoint**，验收标准为：**在 channel 上注入 IPv4/UDP 帧，绑定 echo 端口后能收到回显包（出站队列可观测）**。

| 项 | 说明 |
|----|------|
| 范围 | UDP 头库、端口绑定表、transport demuxer、IPv4 出站封装、UDP echo |
| 链路 | 以 **`channel`** 为主（`InjectInbound` + 出站队列断言）；loopback 作辅助 |
| 验收 | 集成测试 `tests/m1/udp_echo_test.cc`，`ctest -R m1_udp_echo` |
| 下一里程碑 | M2：TCP 单连接简化版（见 [`docs/m2.md`](m2.md)）；M3：TUN demo |

---

## 前置条件（M0 已具备）

实施 M1 前，以下应已稳定（见 [`docs/m0.md`](m0.md) 验收清单）：

- `header/ipv4` + checksum 单测
- `NIC::DeliverNetworkPacket` → `ipv4::Protocol::HandlePacket`（校验和、剥头）
- `link/channel`：`InjectInbound`、出站 `WritePacket` 队列
- `RecordingTransportDispatcher` 或等价 hook（M1 将替换为真实 demuxer）

## 范围边界

### 应包含

| 层 | 内容 |
|----|------|
| `header/udp` | `UDPHeader` 视图、`Encode` / `ParsePorts` / `IsValid`（长度字段） |
| `stack` | `TransportEndpointID`、`TransportProtocol`、`TransportEndpoint`；**`TransportDemuxer`**；`Stack` 注册 UDP、`DeliverTransportPacket` |
| `stack` | **`Stack::AddAddress`**（单 NIC、单 IPv4 地址即可） |
| `ports` | **简化版** `PortManager`：固定端口 bind / 冲突检测；暂不做完整 ephemeral 扫描 |
| `transport/udp` | `Protocol`（`TransportProtocol`）、`Endpoint`（bind + `HandlePacket` + echo 发送） |
| `net/ipv4` | 出站：`SendPacket` / `WritePacket` 路径（构造 IPv4 头 + 调 `LinkEndpoint::WritePacket`） |
| 测试 | channel 注入 → echo → 读取出站队列；UDP 头单测 |

### 明确推迟

| 模块 | 目标阶段 |
|------|----------|
| `waiter`、阻塞 `Read`/`Write` 完整 socket API | M2 或与 TCP 同步引入 |
| UDP 校验和 offload、GSO、`WritePackets` | M1+ |
| 多播/广播 demux、`SO_REUSEPORT`、per-device bind | M1+ |
| ICMP Port Unreachable、分片、ARP、转发、`FindRoute` 复杂逻辑 | M1+ / 扩展 |
| TCP、TUN、iptables | M2 / M3 |

### 与参考实现的差异

| 点 | 参考 (`references/`) | M1 教学栈 |
|----|----------------------|-----------|
| UDP RX 校验和 | 可配置 / 部分路径 offload | 建议 **实现伪首部校验** 或 ADR 写明 M1 先跳过 RX checksum |
| `PortManager` | 完整 ephemeral + reuse | 固定端口表 + `Reserve(port)` 即可 |
| Endpoint | 大状态机 + 收包队列 + waiter | **同步收包队列** + 测试轮询；无 goroutine |
| Demuxer | 多 NIC、raw socket、控制包 | 仅 `(net_proto, trans_proto, local_port)` 精确匹配 |
| `tcpip.Endpoint` | Connect/Listen/Shutdown… | M1 仅需 **`Bind` + echo 回调/内部发送** |

## 推荐实现顺序

自下而上，每步都有可运行测试：

```text
1. header/udp + 单测                    ← ParsePorts、Encode、长度边界
2. TransportEndpointID + 接口骨架       ← registration.hh 扩展
3. TransportDemuxer                     ← Register / Deliver（先 map 查找）
4. Stack 接线                           ← AddTransportProtocol；实现 TransportDispatcher
5. ipv4 出站 Write                      ← 填 Route、写 IPv4 头、LinkEndpoint::WritePacket
6. Stack::AddAddress（最小）            ← NIC 上登记 10.0.0.x，供 local 匹配
7. transport/udp::Endpoint + echo       ← Bind(7)；HandlePacket 回射
8. tests/m1/udp_echo_test.cc            ← channel 往返断言
```

与 [`plan.md`](../plan.md) 教学顺序第 3 步一致：**UDP 端口表 + echo**。

- 出站 IPv4：`network/ipv4` 的 `WritePacket` / endpoint 发送路径

## API 与类型设计

### 1. `TransportEndpointID`（对标 `stack.TransportEndpointID`）

```cpp
struct TransportEndpointID {
  uint16_t local_port{};
  Address local_address;
  uint16_t remote_port{};
  Address remote_address;
};
```

M1 demux 匹配建议：

- **监听 socket**：按 `(local_address, local_port)` 查找；`remote` 为 wildcard（空地址 + 端口 0）。
- **已连接 socket**（M1 可不实现）：四元组全匹配。

### 2. `TransportEndpoint` / `TransportProtocol`

```cpp
class TransportEndpoint {
 public:
  virtual void HandlePacket(Route* r, TransportEndpointID id,
                            PacketBuffer pkt) = 0;
};

class TransportProtocol {
 public:
  virtual TransportProtocolNumber Number() const = 0;
  virtual int MinimumPacketSize() const = 0;
  virtual bool ParsePorts(std::span<const uint8_t> transport_hdr,
                          uint16_t& src, uint16_t& dst) const = 0;
  virtual void HandleUnknownDestinationPacket(Route* r, TransportEndpointID id,
                                              PacketBuffer pkt);
};
```

### 3. `TransportDemuxer`

职责对齐 `stack.transportDemuxer`：

- `RegisterEndpoint(net_proto, trans_proto, id, endpoint*)`
- `DeliverPacket(Route* r, TransportProtocolNumber proto, PacketBuffer pkt)`
  - 调 `TransportProtocol::ParsePorts`
  - 构造 `TransportEndpointID`（local = 目的端口/地址，remote = 源）
  - 命中则 `endpoint->HandlePacket`

由 **`Stack` 实现 `TransportDispatcher`**（替换 M0 每协议手动 `SetTransportDispatcher`）：

```cpp
void Stack::DeliverTransportPacket(Route* r, TransportProtocolNumber p,
                                   PacketBuffer pkt) override;
```

`ipv4::Protocol` 在 `HandlePacket` 末尾仍调用 `transport_dispatcher_->DeliverTransportPacket`；构造 Stack 时将 IPv4 的 dispatcher 设为 `this`。

### 4. `Stack` 扩展

| API | M1 行为 |
|-----|---------|
| `AddTransportProtocol(unique_ptr<TransportProtocol>)` | 注册 UDP(17) |
| `AddAddress(NICID, IPv4Address)` | 记录 NIC 本地地址（单地址即可） |
| `NewUdpEndpoint()` / `Bind(port)` | 返回可 bind 的 endpoint；内部 `demuxer.Register` |
| `GetStats` | 可增加 `udp_*` 计数（可选） |

### 5. `udp::Endpoint`（echo）

最小行为：

1. `Bind(local_port)` → `PortManager::Reserve` + demuxer 注册。
2. `HandlePacket`：从 `pkt.Data()` 读 UDP payload；交换 src/dst 端口与 IP；调用 **出站写路径** 回射。
3. 出站：先 prepend UDP 头，再经 **IPv4 写路径** 交给 `LinkEndpoint::WritePacket`。

M1 可不暴露应用层 `Read()`；测试只断言 **channel 出站队列** 中出现回显帧。

### 6. `ports::PortManager`（简化）

```cpp
class PortManager {
 public:
  bool Reserve(NetworkProtocolNumber net, TransportProtocolNumber trans,
               uint16_t port);
  void Release(...);
};
```

- M1 不做 `16000+` ephemeral 轮询。
- `Bind` 冲突返回 `StackError`（对标 `tcpip.ErrPortInUse`）。

### 7. `Route` 字段填充（入站 / 出站）

入站 `DeliverTransportPacket` 前，IPv4 应在 `Route` 中设置：

- `local_address` = 目的 IP（本机 `AddAddress` 之一）
- `remote_address` = 源 IP
- `net_proto` = IPv4

出站 echo 时反转上述字段，并指定 `NICID`（可从入站 `Route` 或 bind 的 NIC 获得）。

## 出站路径（echo 回复）

```text
udp::Endpoint::SendReply
  → 构造 UDP 头 + payload（vector）
  → ipv4::WritePacket(route, proto=17, pkt)
      → IPv4Header::Encode, checksum
      → nic->GetLinkEndpoint().WritePacket(..., kIPv4ProtocolNumber, pkt)
  → channel 将包放入 outbound 队列（测试读取）
```

**注意**：M1 测试帧仍 **无以太网头**（与 M0 相同）；`WritePacket` 的 `remote`/`local` `LinkAddress` 可为空。

---

## 缓冲区所有权（延续 ADR 001）

- 入站：`HandlePacket` 取得 `PacketBuffer` 所有权；echo 若异步发送须 **拷贝** payload（M1 建议同步发送，避免别名）。
- Demuxer 若向多个 endpoint 复制（M1 无多播），无需 `Clone()`。
- 出站：新建 `vector` 组装 UDP+payload，move 进 `WritePacket`。

建议新增 **`docs/adr/003-m1-udp-echo.md`**：一页写明「无 waiter、测试轮询出站队列、不做 TCP」。

## 测试策略

### 1. `header/udp` 单测

- `Encode` → `SourcePort`/`DestinationPort`/`Length` 一致
- `Length` > 实际 buffer → `IsValid` 失败
- `ParsePorts` 与常量 `kUDPProtocolNumber == 17` 对齐

### 2. Demuxer 单测（可选，推荐）

- 注册端口 `7`，注入伪造 transport 头 + payload，断言唯一 endpoint 收到
- 未注册端口 → `HandleUnknownDestinationPacket` 或丢弃计数

### 3. M1 集成测 `udp_echo_test.cc`

推荐场景（与参考 UDP 测试上下文类似，但无 L2）：

```cpp
// 伪代码结构
Stack s;
s.AddNetworkProtocol(make_ipv4());
s.AddTransportProtocol(make_udp());
auto ch = NewChannel(...);
NICID nic = s.CreateNIC(move(ch));
s.AddAddress(nic, IPv4Address{10,0,0,1});

auto ep = s.NewUdpEndpoint();
ep->Bind(7);  // echo port

std::vector<uint8_t> frame = MakeIpv4Udp(
    /*src*/ {10,0,0,2}, /*dst*/ {10,0,0,1},
    /*sport*/ 12345, /*dport*/ 7,
    payload = "ping");

ch->InjectInbound(kIPv4, frame);

auto outbound = ch->DrainOutbound();  // 需在 channel 上提供测试 API
assert(UdpPayload(outbound) == "ping");
assert(Ipv4Dst(outbound) == {10,0,0,2});
assert(UdpDstPort(outbound) == 12345);
```

**测试辅助**（可放 `tests/m1/packet_builder.hh`）：

- `MakeIpv4Udp(src, dst, sport, dport, payload)` — 正确 IPv4 + UDP checksum（若 M1 启用 UDP RX 校验）
- `ParseIpv4Udp(frame)` — 提取四元组与 payload

### 4. 负向用例（建议同文件）

| 用例 | 期望 |
|------|------|
| 目的端口无 listener | 丢弃；可选 `unknown_port` 统计 |
| UDP `Length` < 8 | `malformed` 递增 |
| 目的 IP 非本机 `AddAddress` | 不 echo（M1 可简化为仍 demux，由测试文档说明） |

## 数据路径（M1）

### 入站

```text
[Test] InjectInbound(IPv4, bytes)
  → NIC::DeliverNetworkPacket
  → ipv4::HandlePacket (校验, 剥 IPv4 头)
  → Stack::DeliverTransportPacket
  → TransportDemuxer::DeliverPacket (ParsePorts)
  → udp::Endpoint::HandlePacket
  → (echo) ipv4::WritePacket → LinkEndpoint::WritePacket
  → channel outbound queue
```

```mermaid
flowchart LR
  Inject[channel InjectInbound]
  NIC[NIC DeliverNetworkPacket]
  IPv4[ipv4 HandlePacket]
  Demux[TransportDemuxer]
  UdpEP[udp Endpoint echo]
  Out[channel outbound]

  Inject --> NIC --> IPv4 --> Demux --> UdpEP --> Out
```

### 出站（echo 回复）

```mermaid
flowchart LR
  EP[udp Endpoint]
  IP4[ipv4 WritePacket]
  Link[LinkEndpoint WritePacket]
  CH[channel queue]

  EP --> IP4 --> Link --> CH
```

## 常见坑

| 坑 | 说明 |
|----|------|
| 忘记 `AddAddress` | demux 或出站不知道本机 IP，回射路由错误 |
| 仍用 per-protocol `RecordingTransportDispatcher` | 集成测通过但 UDP 未接线；应让 `Stack` 统一实现 dispatcher |
| UDP 长度与 IPv4 payload 不一致 | `TotalLength - IHL` 必须 ≥ UDP Length |
| 字节序 | 端口、长度一律 **网络序**（big-endian） |
| 环回双计数 | loopback 自发自收时 RX 统计翻倍；M1 主测 **channel** |
| 过早实现 `tcpip.Endpoint` | 先 echo 再抽象 Read/Write，否则周期膨胀 |
| `transport` ↔ `net` 循环依赖 | 出站写 IPv4 放在 `net/ipv4` 或 `stack` 的独立 `PacketSender` |
